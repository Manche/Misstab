# Misstab

かつて有名だったTwitterクライアントことTweenをモチーフとしたWindows用Misskeyクライアント。
名前のMisstabはシンプルにMisskeyのタブ型クライアントという意味合い。

## 特徴

### 統合タイムライン・結合タイムライン

- 複数サーバのタイムラインを一つに結合！今まではブラウザで複数のタブを使わないと見れなかったのが、一つのタブにまとまって表示！
- もちろん、各タブへの絞り込みや各サーバのみのタイムライン表示も自由自在。

### シンプルなUI

- シンプルにまとまったUIで情報を見逃さない！
- 見やすく分かりやすく素早く操作ができ、あなたのmisskeyライフをサポート！

- .Net Framework 4.8が動作すること

## 元ネタとの違い

### どうしようもないもの

- 超軽量
  - 技術的にスペックの低いPCでは動かないことがあります
- tweenの完全再現(機能的にXとmisskeyに差があるので、UIを完全再現よりはモチーフの体を取っています)
  - 未読管理がない等後述の関係から、UIまわりは若干アレンジしています
- WPF・Electron版
  - そもそもtweenの再現を目的としているので行う予定がありません

### 技術的にちょっと難しい

- fediverseは基本的にリアルタイムなことが多いので、TLの復元や未読管理はありません
- websocketの軽量化(通知情報とタイムラインの取得で1サーバにつき2本以上の接続を行うので、回線がちょっと強くないと辛いです)
- MFM完全再現
  - ライブラリとかコンポーネントください…
- Web版
  - おそらくほぼほぼないかと思います、すでにwebアクセスで揃っているので
  - ただし本家で軽量版のclientがまだないことから、機能を絞った状態での提供は出来るかもしれません
    - ※本家が軽量版を提供するまでの開発となります

### ちょっと面倒
#### 要望次第
- TL更新を一時的に止める
- 色付けを随時変更できる
- リストインポート
  - 有償版対応予定ではあります
- misskey以外のAP対応
  - 今のところmisskey以外にも対応できる構造としていますが、ちょっと手間がかかるかも。
  - 随時forkなどにより機能追加することは可能ですが、正式リリース以後の取り込みになると思います

#### やりません
- **LinuxやMacOSへの対応**
  - 一度検討しましたが、XAML化が必要だったりバグが起きる原因になるので行わないことにしました
  - 必要であればforkしてください。当方ではサポートしません。
    - もしforkした場合はMisstab以外の名称を付けてください
    - 同じwindows版開発の際はご一報いただけると嬉しいかも
- AI入出力対応
